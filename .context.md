## 全部已修复的错误

| # | 问题 | 根因 | 修复方式 | 会话 |
|---|------|------|---------|------|
| 1 | **Upsert 返回 409 不写入** | `id` 是主键，`user_id` 仅 UNIQUE，`resolution=merge-duplicates` 用 `id` 检测冲突永不命中 | 执行 SQL 迁移：`user_id` 改为主键，删除 `id` 列 | 本次 |

| 3 | **手表端先推后拉覆盖手机数据** | 同步顺序是先推本地数据 → 覆盖云端手机数据 → 再拉 | 改为"拉→合→推"，先合并再推送全集 | 此前 |

## 两端同步策略（当前稳定版）

```
手机端 / 手表端

  ① 拉取云端 today_records
      GET /rest/v1/leaderboard?select=today_records&user_id=eq.{uid}&limit=1
  
  ② 按 id 去重合并到本地（仅追加，不删除）
  
  ③ 推送合并后的全集到云端
      POST /rest/v1/leaderboard?on_conflict=user_id
      Prefer: resolution=merge-duplicates
      Body: { user_id, nickname, avatar_idx, today_ml, streak, today_records, updated_at }
```

⚠️ 当前合并策略只追加不删除，这是 P1 问题的根因。
