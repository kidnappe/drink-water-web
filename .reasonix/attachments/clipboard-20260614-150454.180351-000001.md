## 来自 Web 版助手的手札 —— 数据同步与设备配对

你好，Web 版现在已完成双向同步和设备配对功能。手表版需要对等实现以下功能才能互通。

---

### 一、数据库表结构

两个表，均在 Supabase 项目中（与排行榜共用同一个项目）：

**leaderboard 表（已有，新增 today_records 字段）**
```sql
user_id TEXT PRIMARY KEY
nickname TEXT DEFAULT ''
avatar_idx INTEGER DEFAULT 0
today_ml INTEGER DEFAULT 0
streak INTEGER DEFAULT 0
today_records JSONB DEFAULT '[]'::jsonb   -- [{id, amount, time, mood}]
updated_at TIMESTAMPTZ DEFAULT NOW()
```

**device_pairing 表（新建）**
```sql
pairing_code TEXT PRIMARY KEY      -- 6 位数字配对码
user_id TEXT NOT NULL               -- Web 版的 user_id
created_at TIMESTAMPTZ DEFAULT NOW()
```

### 二、手表版需要实现的功能

#### 2.1 Supabase 配置界面

手表版需要有一个"在线排行榜"设置入口，与 Web 版相同的配置项：

- Supabase URL
- Anon Key
- 测试连接按钮
- 连接状态显示

**Supabase JS SDK（CDN 引入）** 或使用 **fetch REST API** 均可。如果手表版之前使用 fetch REST API，注意端点格式：

```javascript
// fetch REST API 方式调用 leaderboard upsert
fetch(supabaseUrl + '/rest/v1/leaderboard', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'apikey': supabaseAnonKey,
    'Authorization': 'Bearer ' + supabaseAnonKey,
    'Prefer': 'resolution=merge-duplicates'

  body: JSON.stringify({
    user_id: userId,
    nickname: nickname,
    avatar_idx: avatarIdx,
    today_ml: todayTotal,
    streak: streak,
    today_records: todayRecords,
    updated_at: new Date().toISOString()
  })
});
```

配对成功后，手表和手机使用同一个 `user_id`，后续所有 `syncTodayToLeaderboard()` 操作都自动合并到同一个人。

**建议配对手表 UI：**
- 设置页新增"连接手机"菜单项

- 确认按钮 + 加载中状态
- 成功后提示"已与手机同步"

#### 2.5 强制同步按钮

```javascript
function forceSync() {
  syncTodayToLeaderboard()     // 先推送本地数据到云端
    .then(mergeCloudTodayRecords)  // 再拉取云端数据合并到本地
    .then(function(count) {
      refreshUI();
      // 提示"同步完成，合并了 X 条记录"
    });
}
```

在设置页添加"立即同步"菜单项。

### 四、需要手表版注意的点

1. **record id 必须保留** — Web 版使用 `crypto.randomUUID()` 生成 id，手表版也要生成并在推送时带上
2. **`syncTodayToLeaderboard()` 推送的 today_records 字段中**，每个元素必须包含 `id`、`amount`、`time`、`mood`
3. **合并时不要覆盖本地记录** — 以 id 去重，只有本地没有的才追加
4. **深色模式、localStorage key 等与 Web 版保持一致**（数据兼容）
5. **排行榜拉取现在按需加载明细** — `fetchOnlineLeaderboard()` 不再拉 `today_records`，只在用户点击详情时才拉
