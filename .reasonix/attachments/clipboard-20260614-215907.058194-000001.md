## 给手表版：推送逻辑排查与修复

手表能拉取云端数据（能看到手机的记录），但推不上去，手机端查不到手表写的任何数据。

---

### 一、手表端需要加的调试面板

在**设置页**的"立即同步"按钮下方，加一个调试区域，与手机 Web 版保持一致：

**HTML 结构：**
```html
<div id="syncDebug" style="display:none;font-size:0.75rem;color:#666;
     background:#f8f9fa;border-radius:8px;padding:8px 10px;margin:4px 0;
     line-height:1.6;word-break:break-all;"></div>
```

**JS 函数：**
```javascript
function syncDebug(msg) {
  var el = document.getElementById('syncDebug');
  if (!el) return;
  el.style.display = 'block';
  el.innerHTML = el.innerHTML + '<div>' + msg + '</div>';
  el.scrollTop = el.scrollHeight;
}
function syncDebugClear() {
  var el = document.getElementById('syncDebug');
  if (el) { el.innerHTML = ''; el.style.display = 'none'; }
}
```

然后在 `forceSync()` 或 `syncTodayToLeaderboard()` 中调用 `syncDebug('...')` 输出每一步的日志。

---

### 二、确认 `syncTodayToLeaderboard` 被调用

在 `addWater()`（喝水记录）和 `deleteRecord()`（删除记录）末尾必须调用：

```javascript
function addWater(amount) {
  // ... 保存本地记录 ...
  syncTodayToLeaderboard();  // ← 必须有
}

function deleteRecord(id) {
  // ... 删除本地记录 ...
  syncTodayToLeaderboard(true);  // ← 删除时传 true，跳过拉取合并
}
```

### 三、推送函数中加入详细日志

```javascript
function syncTodayToLeaderboard(skipPull) {
  syncDebug('📤 开始推送数据到云端...');
  
  var uid = localStorage.getItem('drink_water_user_id');
  syncDebug('user_id: ' + (uid ? uid.substring(0, 8) + '...' : '未设置'));
  
  if (!uid) { syncDebug('❌ user_id 为空，无法推送'); return; }

  // 构建推送数据
  var todayKey = getDateKey(new Date());
  var todayRecords = gState.records.filter(r => r.dateKey === todayKey).map(r => ({
    id: r.id,
    amount: r.amount,
    time: r.dateTime,
    mood: r.mood || ''
  }));
  
  var body = {
    user_id: uid,
    nickname: me.name,
    avatar_idx: me.idx,
    today_ml: getTodayTotal(gState.records),
    streak: calcStreak(gState.records, goal),
    today_records: todayRecords,
    updated_at: new Date().toISOString()
  };
  
  syncDebug('📦 推送 ' + todayRecords.length + ' 条记录, 总计 ' + body.today_ml + 'ml');

  fetch(supabaseUrl + '/rest/v1/leaderboard', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'apikey': supabaseAnonKey,
      'Authorization': 'Bearer ' + supabaseAnonKey,
      'Prefer': 'resolution=merge-duplicates'
    },
    body: JSON.stringify(body)
  }).then(function(resp) {
    syncDebug('🌐 HTTP 状态码: ' + resp.status);
    if (resp.status === 200 || resp.status === 201) {
      syncDebug('✅ 推送成功');
    } else if (resp.status === 409) {
      syncDebug('⚠️ 数据已存在（409，属于正常情况）');
    } else {
      syncDebug('❌ 推送失败，状态码: ' + resp.status);
    }
    return resp.text();
  }).then(function(text) {
    if (text && text.length < 200) syncDebug('响应: ' + text);
  }).catch(function(err) {
    syncDebug('❌ 网络请求失败: ' + (err.message || err));
  });
}
```

### 四、重点检查项

| 检查项 | 正确值 | 错误后果 |
|--------|--------|---------|
| Supabase URL | 与手机端**完全一致**（注意末尾无空格） | 请求发到不同服务器 |
| Anon Key | 与手机端完全一致 | 认证失败，返回 401 |
| `user_id` | 从 `localStorage.drink_water_user_id` 读取 | 推送到不同行，数据不合并 |
| 请求方法 | **`POST`**（不是 `PUT`、`PATCH`） | 不触发 upsert |
| 请求头 | `Prefer: resolution=merge-duplicates` | 重复数据报错或覆盖 |
| `today_records` 格式 | `[{id, amount, time, mood}]`，`id` 必须唯一 | 去重失败，数据重复 |
| 表名 | `leaderboard` | 找不到表，返回 404 |
| 列名 | `user_id`, `nickname`, `avatar_idx`, `today_ml`, `streak`, `today_records`, `updated_at` | 字段名对不上，返回 400 |
