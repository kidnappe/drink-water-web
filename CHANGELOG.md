## v1.4.0 — 日历交互 + 深色模式 + 排行榜周期

### ✨ 新增功能

- **📅 日历日期可点** — 点击热力图中任一天，弹窗查看当日每条喝水记录，记录过多时自动滚动
- **📅 日历翻月** — 日历标题栏新增 ◀ ▶ 按钮切换月份，切回历史页时自动重置到当前月
- **👅 喝一小口 (15ml)** — 首页水量按钮下方新增「喝一小口」按钮，适合抿一口就记录
- **🏆 排行榜本周/本月** — 数据库新增 `week_ml` / `month_ml` 字段，切换标签时正确显示周期累计排名
- **📋 关于页更新日志** — 关于应用弹窗内新增滚动版更新日志，所有历史版本一览
- **🌙 深色模式完整样式** — 补全所有页面元素深色配色（卡片、导航、弹窗、图表等）
- **🌙 auto 按钮** — 深色模式标签旁新增 `auto` 按钮，一键切回跟随系统
- **⏱ 提醒周期可调** — 新增「提醒周期」选项，可自定义 1-60 分钟，默认 30 分钟

### 🐛 修复

- **深色模式无效果** — 补全了完整的深色模式 CSS，覆盖所有页面元素
- **自定义按钮被挤到第二行** — 缩小按钮最小宽度，5 个按钮始终在同一行
- **刷新后通知不恢复** — 页面加载时若通知权限为未决状态，主动请求授权后重启定时器
- **排行榜切换标签排名不变** — 改用对应周期的累计字段排序和展示

### 🎨 优化

- **排行榜去掉 10 秒自动轮询** — 只在切到排行榜页或手动刷新时拉取，不再后台定时请求
- **累计值缓存** — `addWater`/`deleteRecord` 时增量更新今日/本周/本月累计值，避免每次遍历全部历史记录
- **攒批推送** — 连续喝水时攒 30 秒一批推送到云端，不多次 HTTP 请求
- **撒花动画减半** — 粒子数从 80 降到 40，加冷却锁防止连续触发堆积 DOM
- **导出数据修复** — 导出后的临时内存链接确保释放，避免数据堆积后内存泄漏

### 🔧 技术改动

- **深色模式改为三态** — 跟随系统 / 开启 / 关闭，开关仅切换开/关，auto 按钮独立控制
- **数据库迁移** — `leaderboard` 表新增 `week_ml` / `month_ml` 字段，见 `migrate-week-month-ml.sql`
- **推送逻辑更新** — `syncTodayToLeaderboard()` 推送时同步计算并上传周/月累计值
- **CSS 变量化** — 品牌色 `#4A90E2` 等定义在 `:root`，后续改主题色只改一处
- **localStorage key 常量化** — 所有存取 key 统一收到 `LS` 对象，手误拼错会直接报错
- **DOM 查询缓存** — 新增 `$()` 辅助函数，同一个元素只查一次
- **连续达标天数优化** — 先建日期映射表再计算，避免循环内反复遍历全部记录
- **XSS 防范** — 用户名不再拼进 HTML onclick，改用 `data-` 属性 + 事件委托
- **应用图标更新** — 改用 SVG 矢量图标（蓝色水杯），manifest.json 和 favicon 同步更新

## v1.3.0 — 零外部依赖 + 同步稳定

### 🐛 修复

- **Safari "无法打开网页，丢失网络连接"** — 移除了 Supabase JS SDK 和 Chart.js 两个外部 CDN 依赖，应用零外部资源加载，不会被 Safari 内容拦截器阻拦
- **Upsert 返回 409 不写入** — `leaderboard` 表主键从 `id` 改为 `user_id`，`resolution=merge-duplicates` 能正确检测冲突合并数据

### 🔧 技术改动

- **移除 Supabase JS SDK** — 全部 8 处 API 调用改为原生 `fetch`（与手表端统一使用 `sbFetch` 辅助函数）
- **移除 Chart.js** — 历史图表改用原生 Canvas API 自绘（柱状图 + 目标红线 + 达标绿色标记）
- **移除 Service Worker** — 删除了 `sw.js` 及其注册代码
- **清理遗留代码** — 删除了 `_sbClient`、`getSupabaseClient()` 等 SDK 相关残留
